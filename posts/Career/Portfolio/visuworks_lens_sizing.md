## 경험에 의존하던 사이즈 선택 문제에 예측 기반 보조지표를 제공해 수술 안정성 10% 향상

### 문제

시력교정 렌즈 삽입술에서 렌즈 사이즈 선택은 수술 경과를 결정하는 가장 중요한 요인 중 하나다. 하지만 어떤 공식이 있는 게 아니라 의사들의 경험에 의존해 결정되고 있었다. 의사마다 기준이 다르고, 시간에 따라 휴리스틱도 바뀌는 상황이었다.

### 해결

수술 후 경과를 판단하는 3가지 핵심 지표(Vault, 전방각, 렌즈 회전)를 수술 전에 예측해 제공하는 AI 서비스를 개발했다. 의사가 각 렌즈 사이즈별 예측 결과를 비교해 최적의 사이즈를 판단할 수 있도록 했다.

### 성과

서비스 도입 전후 수술 결과를 비교 분석한 결과:
- **Vault 안정 비율** (250~750μm 범위 내): 73% → 83% (+10%p)
- **월별 렌즈 재위치 건수**: 20건 → 10건 (-50%)

## 단순 예측값의 한계를 Quantile Regression 기반 확률 분포 추정으로 해결, 렌즈 위치·각도·회전 안정성 3개 지표의 수술 위험도 정량화 파이프라인 구축

### 문제

모델 고도화를 진행했지만 MAE 110, R² 0.4 수준에서 성능이 더 이상 오르지 않았다. 원인을 분석한 결과, 렌즈 종류(Model Name)가 Vault에 결정적인 영향을 미치는데 현재 데이터에서는 이 정보를 정확히 파악할 수 없었다. 즉, 데이터 자체에 내재된 불확실성(Aleatoric Uncertainty) 때문에 점 추정의 한계가 명확했다.

### 해결

문제를 재정의했다. "정확한 값을 맞추는 것"이 불가능하다면, "불확실성을 포함한 예측 구간"을 제공하는 것이 더 현실적이라고 판단했다.

- **Quantile Regression** 도입: 평균이 아닌 분위수(5%, 95% 등)를 추정해 예측 분포를 산출
- **Conformal Prediction(CQR)** 적용: Calibration Set을 활용해 목표 신뢰수준(예: 90%)을 통계적으로 보장하는 예측 구간 생성
- **평가지표 재설계**: MAE 대신 포함률(Coverage)과 구간 길이(Interval Width)를 핵심 지표로 정의

### 성과

- Vault, 전방각, 렌즈 회전 3개 지표 각각에 대해 "정상 범위 내 위치 확률"을 직접 제시하는 파이프라인 구축
- 예: "예측값 520μm, 정상 범위(250~750μm) 내 위치 확률 91%"

## 블랙박스 모델의 낮은 현장 신뢰도를 불확실성 추정·확률 표현·의학적 직관 기반 지표 설계로 해결, 현장 사용자 19명 대상 신뢰도 50% 향상 (2.8 → 4.2점)

### 문제

기존 서비스의 현장 신뢰도 평가 결과, 5점 만점에 2.8점으로 낮았다. 주요 피드백:
1. **불확실성 정보 부재**: "이 숫자가 얼마나 확실한 건지 모르겠다"
2. **출력 형식의 직관성 부족**: "어디에 집중해야 할지 모르겠다"
3. **책임 소재 불명확**: "AI가 값을 딱 제시하면 내가 그걸 따른 건지 아닌지 경계가 모호하다"

### 해결

두 가지 축에서 서비스를 전면 개선했다.

**1. 예측 방식 고도화**
- 단일 예측값 → 분위수 회귀 기반 확률적 출력 구조로 전환
- 임상적으로 유의미한 기준값을 기준으로 "정상 범위 내 위치 확률"을 직접 제시
- 의사가 확률적 판단을 직접 내릴 수 있는 구조로 설계

**2. UX/표현 방식 설계**
- 확률 분포를 시각적 그래프로 표현
- 위험도를 색상 기반 등급(녹색/주황/적색)으로 표현
- 3가지 지표를 한 화면에서 파악할 수 있는 레이아웃 적용

### 성과

동일한 19명(의사 11명, 검안사 8명) 대상 사전-사후 비교:

| 평가 축 | 기존 | 개선 | 개선폭 |
|---------|------|------|--------|
| 서비스 신뢰도 | 2.8 | 4.2 | +1.4 |
| 정보 이해 용이성 | 2.3 | 4.5 | +2.2 |
| 임상 활용성 | 2.5 | 4.0 | +1.5 |
| 의사결정 보조 효과 | 2.4 | 3.9 | +1.5 |
| 지속 사용 의향 | 2.6 | 4.1 | +1.5 |

## 피처 엔지니어링 및 모델 튜닝으로 기존 베이스라인 대비 예측 성능 30% 향상

### 문제

기존 모델의 성능을 확인하려 했으나 여러 문제가 있었다:
- 추론 로그와 모니터링 기능 부재로 실제 성능 확인 불가
- 학습/추론 상황에서 데이터 전처리 로직이 달라 일관성 문제
- 실험이 notebook 형태로만 존재해 형상관리와 추적 어려움

### 해결

**1. ML 시스템 구축**
- 데이터 파이프라인과 Feature Store 구현으로 학습-추론 데이터 일관성 확보
- 모니터링 시스템 구축 (성능 추적, Data Drift 감지, 자동 알림)
- Hydra + W&B로 실험 관리 체계화

**2. 도메인 지식 기반 피처 개발**
- 도메인 전문가와 협업해 가설 설정 → 데이터 검증 → 피처화 반복
- 예: 나이-수정체 두께 관계 가설 검증 후 피처로 활용

**3. 신뢰도 평가지표 개발**
- 예측값이 의학적 직관과 일치하는지 평가하는 지표 설계
- 물리적 타당성, 유사 환자군 일관성 등 종합 평가

### 성과

- 데이터 불일치로 인한 성능 저하 문제 해결 (약 20% 개선)
- 도메인 피처 및 튜닝으로 기존 베이스라인 대비 **예측 성능 30% 향상**
- Silent failure 사전 감지 가능한 모니터링 체계 구축
