## 수동 조회 프로세스를 OCR 파이프라인으로 자동화, 데이터 확인 시간 95% 단축 (20초 → 1초/건)

### 문제

병원 검사 결과 이미지에서 데이터를 추출해 데이터베이스에 적재해야 했다. 기존에는 담당자가 검사 결과 이미지를 수동으로 조회하고 입력하는 방식이었고, 건당 약 20초가 소요되었다. "시력 교정 수술 추천 서비스"나 "렌즈 사이즈 추천 서비스"를 안정적으로 운영하려면 실시간으로 정확한 데이터 수집이 필수였다.

### 해결

6가지 종류, 총 40~50대 검사장비에서 이미지를 실시간으로 수신해 OCR 처리 후 DB에 적재하는 파이프라인을 구축했다.

**1. 비동기 처리 구조 도입**
- Client-Server 구조를 재정의: OCR 처리 지연이 사용자 경험에 영향을 주지 않도록 설계
- Client는 이미지 송수신 성공 여부만 확인, OCR은 Server에서 비동기 처리
- 평균 응답 속도 5초 → 1초로 단축

**2. 코드 구조 개선**
- 절차지향적 레거시 코드를 객체지향 구조로 재설계
- 기능과 역할을 명확히 분리해 에러 추적 용이하게 개선
- pytest 기반 unit test와 mypy 타입 검증을 CI에 통합

### 성과

- 데이터 확인 시간 **95% 단축** (20초 → 1초/건)
- 안정적인 데이터 수집 기반 구축으로 서비스 운영 안정성 확보

## 고정 레이아웃 특성을 활용해 Text Detection은 룰베이스, Text Recognition은 TrOCR로 설계, 검증 세트 기준 Character-level Accuracy 100% 달성

### 문제

의료 서비스 특성상 OCR 정확도가 특히 중요했다. 목표는 99% 이상이었으나, 초기에는 평가용 데이터셋 자체가 없어 기존 모델 성능을 확인할 수 없었다. 실제 검사 이미지 약 200건으로 테스트셋을 구축해 평가한 결과, 기존 모델은 목표에 미치지 못했다.

### 해결

문제의 특성을 분석해 효율적인 접근을 설계했다:

**1. Text Detection 단계 단순화**
- 검사 결과 이미지가 고정된 레이아웃이라는 특성 활용
- 텍스트 위치를 사전에 정의해 별도 Detection 모델 불필요

**2. Text Recognition 특화 모델 도입**
- TrOCR 오픈소스 모델 도입 (Text Recognition에 특화된 모델)
- 이미지 전처리 고도화 및 결과 후처리 개선 반복

**3. 모델 서빙 최적화**
- TritonServer로 온프레미스 배포
- 동적 배치 처리와 동시성 관리로 처리량 확보

### 성과

- 검증 세트(200건) 기준 **Character-level Accuracy 100%** 달성
- 정확도 99%, 에러율 1% 미만의 안정적인 파이프라인 구축
