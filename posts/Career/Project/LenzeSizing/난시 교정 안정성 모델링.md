토릭 (Toric) -> 난시 교정용 렌즈를 뜻하는 것.

난시 교정 안정성이라고 표현하는게 이해는 더 쉬울 듯.

데이터 파이프라인 설계는 
- 피처 데이터 :  ( OCR 을 통해 정리했던 데이터 베이스 )
- 결과 : 수술 후 렌즈 각도

피처 데이터는 기존에 정리해놨던 파이프라인을 그대로 사용했고, 레이블은 새롭게 추출했음.

OPD Scan라는 장비에서 사람이 수작업으로 렌즈의 위치를 표기해놓은 이미지가 있는데, 여기에서 렌즈가 자리잡은 위치를 확인할 수 있음.
( 추출은 OpenCV로 선 추출 -> 선의 기울기 추출 -> 각도로 변환 )

---

데이터의 개수는 3000개 정도 ( 2022 ~ 2026 )
생각보다 전처리하는 과정에서 날라가는게 많았음.

레이블을 어떻게 정할지가 고민했던 것들 중 하난데
- axis : 렌즈의 난시교정축
- target axis : 사람이 가지고 있는 난시축
- 수술 후 렌즈각도 (수술결과)

난시교정수술, 그 중에서 렌즈 삽입술에서 중요한 건 목표한 위치에 렌즈가 고정되어 있는지.
( 사람의 난시축과 렌즈의 난시축이 다르면 돌려서 넣게 됨 )
돌아갔다면 난시축이 바뀌니까 사람이 불편함을 크게 느낌.

[[난시 교정 안정성]]
> 난시축이 안맞으면 같은 도수더라도 불편함에 큰 영향을 주게 됨.


따라서 수술하기 전에 렌즈가 돌아갈 것인지를 예상할 수 있으면 수술하는데 큰 도움이 된다.
이런 목적에 맞게 모델을 학습시키기 위해서는 "렌즈 회전여부 혹은 회전각도"가 레이블이 되어야함.

target_axis가 80도고 axis가 70도면 반시계방향으로 10도를 돌려서 넣음.
- 결과가 +10도면 -> 렌즈가 고정 -> 회전각도 0
- 결과가 -5도면 -> 회전각도 15도

따라서 다음처럼 정의할 수 있다
- target angle = target axis - axis
- rotation angle = abs(target angle - result angle)

회전각도를 레이블로 정의했을 때는 이렇게 풀어볼 수 있겠다.

---

모델링 결과는 조금 대참사...

$R^2$ 가 0.02정도 나오네. (선형모델), 트리모델로 하면 음수 찍고.

더 큰 문제는 내가 예상했던 패턴이 없다.

> 렌즈 사이즈가 커지면 -> 토릭 안정성 증가

이게 기본적으로 생각했던 모델링 컨셉이고 의학적인 직관으로 이해하고 있는데,
사이즈와 회전 각도 간의 상관관계는 거의 0이고, 모델도 이걸로 패턴을 잡고 있지 않다.

1차 실험은 실패로 보이는데, 어떻게 해결해나가야할까??
