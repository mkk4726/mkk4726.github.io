## 개요

T-ICL 수술 예측을 위해 서로 다른 임상 타겟을 가진 **3개의 독립된 모델**을 구성했다. 타겟에 따라 피처 전략도 달라진다.

| 모델 | Y (예측 타겟) | 핵심 메커니즘 |
| --- | --- | --- |
| Vault | 수술 후 vault 높이 | ICL ↔ 수정체 사이 거리 → **전후 공간 여유** |
| Angle | 수술 후 iridocorneal angle | 홍채 ↔ 각막 사이 각도 → **전방각 구조** |
| Toric Stability | 수술 후 axis 변화량 | 렌즈 회전 저항 → **고정력 및 마찰** |

---

## 모델 선택 — GAM (Generalized Additive Model)

### 선택 이유

모델링에서 가장 중요하게 생각한 것은 **모델이 찾아낸 패턴을 직접 확인하고, 이를 의학적 패턴과 비교·검증하는 과정**이었다.

이 관점에서 GAM이 가장 적합하다고 판단한 이유:

- **X → Y 사이의 비선형 관계를 시각화**할 수 있음 (partial dependence plot)
- 원하는 방향으로 모델 구조를 직접 설계할 수 있음
- 임상 전문가와 결과를 공유하고 피드백받기 용이함
- 블랙박스 모델 대비 "왜 이 예측인가"를 설명할 수 있음

### 성능 향상 요약

- Baseline MAE: **140**
- 최종 MAE: **100 초반**
- 기여 요인
  1. 데이터 정제 및 파이프라인
  2. 도메인 기반 피처 엔지니어링
  3. 모델 튜닝

## 피처 엔지니어링

### 핵심 아이디어

T-ICL은 홍채 뒤쪽 **posterior chamber (후방부)** 에 삽입된다. 따라서 렌즈가 실제로 안착하는 **후방부 공간의 구조**를 다각도로 근사하는 것이 핵심.

**데이터 환경**

- 직접 지표 (CASIA OCT 기반): CLR (Crystalline Lens Rise), ACD, 각막 두께, 각막 곡률(K)
- 간접 추정 지표: WTW, 렌즈 사이즈, 근시 도수 (STS 직접 측정 없음)

> STS (Sulcus-to-Sulcus) 직접 측정값이 없어 WTW 및 근시 도수 기반으로 후방 폭을 간접 추정했다.

---

### Vault 모델

수술 후 vault = ICL이 수정체 위에 얼마나 높이 떠 있는가. **후방부 공간 여유**가 핵심.

| 피처                          | 반영하는 생리적 의미                                    |
| --------------------------- | ---------------------------------------------- |
| CLR (Crystalline Lens Rise) | 수정체가 홍채면 기준으로 얼마나 앞으로 돌출했는가 — 후방 공간 여유의 직접 지표  |
| CLR / ACD                   | 전방 깊이 대비 후방부 수정체 돌출 비율 — 후방 공간이 전체에서 얼마나 잠식됐는지 |
| 렌즈 사이즈 / 눈 길이 (비율·차이)       | ICL이 실제로 fit 되는 수평 공간 여유                       |
| 나이 기반 수정체 두께 그룹             | 나이 들수록 수정체 두꺼워지고 CLR 증가 → vault 공간 감소          |
| (WTW − ICL size) / ACD      | 수평 공간 대비 전방 깊이의 상대적 여유                         |
| 고도근시 + 난시 조합                | 안축장 길이 proxy → 눈 크기 추정                         |

> 고도근시일수록 안구가 커지고 안축장이 길어지는 경향이 있어, STS 직접 측정이 없을 때 근시 도수를 간접 proxy로 활용. CLR은 CASIA OCT로 직접 측정 가능한 후방부의 핵심 지표.

---

### Angle 모델

수술 후 iridocorneal angle = 홍채와 각막이 이루는 각도. 전방각 구조에 의해 결정된다.

| 피처 | 반영하는 생리적 의미 |
| --- | --- |
| ACD / 각막 곡률(K) | 전방 깊이 대비 각막 커브 — 각도에 직접 영향 |
| ACD × WTW | 전방각의 기하학적 근사 (삼각형의 두 변에 해당) |
| 근시 도수 | 고도근시일수록 안구 확장 → 전방각 구조 변화 |
| 홍채 두께 / 위치 (있는 경우) | 가장 직접적인 각도 결정 인자 |

> 전방각은 수술 전 구조가 아닌 **수술 후** 값을 예측하는 것이므로, 렌즈 삽입으로 인한 후방부 압력 변화가 홍채 위치에 미치는 영향도 고려.

---

### Toric Stability 모델

수술 후 axis 변화량 = 렌즈가 얼마나 회전했는가. 마찰·고정력이 핵심.

| 피처 | 반영하는 생리적 의미 |
| --- | --- |
| WTW − ICL size (절댓값) | 작을수록 snug하게 고정 → 회전 적음 |
| 초기 정렬 오차 (목표 axis vs 삽입 axis 차이) | 초기 오차가 클수록 이후 회전 가능성 ↑ |
| 난시 도수 크기 | 토크 크기에 비례 — 도수가 클수록 렌즈에 걸리는 힘 ↑ |
| 난시 도수 / 구면 도수 비율 | 난시 비중이 높을수록 toric 렌즈에 걸리는 토크 달라짐 |
| 수술 후 초기 측정 시점 | 부종 등 초기 변화가 측정값에 섞여 있을 수 있음 |

> 회전 안정성은 단순히 렌즈 크기만의 문제가 아니라, **초기 정렬 정확도 + 눈과 렌즈의 물리적 fit** 이 복합적으로 작용한다는 점이 핵심 인사이트.

## 추가 피처 아이디어 (검토 대상)

**안축장 proxy 강화**

- `근시 도수 × 각막 곡률(K-value)` 조합 → 안축장 직접 측정이 없을 때 간접 추정 가능. Holladay 공식 계열에서 사용하는 접근법.

**Vault 모델**

- `(WTW − ICL size) / ACD` — 단순 차이보다 상대적 여유를 더 잘 반영
- `CLR / 수정체 두께` — 수정체 두께 대비 돌출 정도, 나이 효과를 부분적으로 통제

**Toric Stability 모델**

- `cos(2 × axis 오차)` 변환 — 각도는 180° 주기이므로 선형 차이보다 cosine 변환이 더 적합할 수 있음

## 회고 및 포폴 표현 방향

### 현재 포폴의 약점

- 성능 수치(MAE 140 → 100 초반)는 있지만 **향상 과정의 논리**가 빠져 있음
- 피처 엔지니어링이 "도메인 지식 기반"이라고만 적혀 있고 **구체적 근거**가 없음

### 보완 방향

1. **왜 이 피처인가** — 각 피처의 임상적 의미를 한 줄씩 연결
2. **모델이 찾은 패턴을 어떻게 검증했는가** — GAM partial plot에서 발견한 비선형 관계와 의학적 해석 연결
3. **Ablation 또는 피처 중요도** — 피처 추가 전후 성능 변화로 설득력 강화
4. **3개 모델 간의 차별화** — 같은 GAM이지만 각 모델이 어떤 다른 문제를 풀고 있는지 명확히

## 성능 plateau의 원인 분석 — "렌즈 정보의 부재"

여러 모델을 비교하고 튜닝을 진행하는 과정에서 **MAE 110, R² 0.4** 수준에서 성능이 더 이상 오르지 않는 한계에 부딪혔습니다. 데이터 파이프라인을 구축하고, 도메인 지식을 활용해 피처를 개발하고, 하이퍼파라미터를 튜닝하며 집요하게 성능 향상을 시도했지만 역부족이었습니다.

**"왜 성능이 오르지 않을까?"** 라는 질문을 끝까지 파고들었고, 결국 **"렌즈 정보의 부재"**가 원인임을 밝혀냈습니다. 수소문 끝에 렌즈 제조사에 문의한 결과, **렌즈 종류별로 기본 Vault(Base Vault)가 다르다**는 사실을 확인했습니다. 즉, 어떤 렌즈를 썼는지에 대한 정보 없이는 정확한 예측이 불가능했던 것입니다. 하지만 현재 데이터셋은 렌즈 종류가 수기로 입력되어 결측이나 오기입이 많아, 이를 모델링에 활용하기 어려운 상황이었습니다.

이러한 데이터의 한계를 명확히 인정한 후, 전략을 수정했습니다. "정확한 값을 맞추는 것(Point Prediction)"이 불가능하다면, **"불확실성을 포함한 예측 구간(Prediction Interval)"을 제공**하여 의사의 의사결정을 돕는 것이 더 현실적이고 유용한 해결책이라고 판단했습니다.

→ 이후 전개 과정은 [[Career/Project/LensSizing/Quantile Regression 및 CQR 파이프라인|Quantile Regression 및 CQR 파이프라인]] 참고.

## Data Leakage 방지

실험 과정에서 가장 주의한 점은 data leakage입니다.

데이터 자체가 모델에 직접 노출되지 않도록 주의했을 뿐 아니라, test set을 반복적으로 확인함으로써 hyperparameter가 과적합되는 문제도 방지해야 했습니다. 이를 위해 실험 주기를 사전에 정의하고, 배포 직전에만 test set을 검증하는 방식으로 진행했습니다.

이러한 엄격한 검증 프로세스를 통해 실험 환경에서의 성능과 실제 서빙 환경에서의 성능 차이를 최소화할 수 있었습니다.

## 신뢰도 평가지표 개발

단순 성능 지표(MAE, R²)만으로는 서비스의 신뢰도를 평가하기 어렵다는 문제를 인식했습니다.

예측 결과가 도메인 지식 및 직관과 얼마나 일치하는지를 평가할 수 있는 신뢰도 지표를 새롭게 개발했습니다. 이 지표는 예측값이 물리적으로 타당한 범위 내에 있는지(예: 렌즈 사이즈가 커질수록 Vault 예측값이 커지는지), 유사한 환자군에서 일관된 예측을 하는지 등을 종합적으로 평가합니다.

개발한 신뢰도 지표는 팀 내에서 채택되어 모델 평가의 표준 지표로 사용되고 있으며, 사용자의 신뢰도를 확보하는 데 기여했습니다.
