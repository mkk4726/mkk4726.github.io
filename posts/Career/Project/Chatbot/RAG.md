**모델링 — 병원 도메인 특화 RAG 파이프라인 개선**

- 병원 관련 질의응답 챗봇의 기본 RAG 파이프라인에서 검색 누락 및 부정확한 답변 문제 발생
- 도메인 특화 파이프라인 설계 및 적용:
    - **키워드 추출 기반 하이브리드 검색** — 의료 용어·병원 특화 키워드를 추출해 BM25 키워드 검색과 벡터 검색을 결합, 임베딩 모델의 도메인 어휘 취약점 보완
    - **질문 재정의 (히스토리 반영)** — 다턴 대화에서 이전 맥락을 반영해 질문을 독립적인 형태로 재구성, 문맥 단절로 인한 검색 오류 제거
    - **질문 분해 (Query Decomposition)** — 복합 질의를 단위 질문으로 분리 후 통합, 다중 의도 질문의 검색 커버리지 향상
- 검색 성능 **Hit Rate 기준 70% 향상**
- LLM 생성 후 직접 검수한 평가셋(golden set) 기반으로 답변 정확도 **40~50점 → 90~100점** 향상

**다국어 확장 — Agentic RAG 도입**

서비스 다국어 확장(영어·중국어·태국어·일본어 등) 과정에서 한국어 특화 파이프라인의 한계를 진단하고, 언어별 검색 전략을 분리 설계

- **문제** — 기존 파이프라인은 한국어 문서 기반의 키워드 추출·Query Rewriting에 특화되어 있어, 다국어 환경에서 검색 품질 보장이 어렵고 검증·유지보수 비용이 높음
- **설계 결정** — 한국어는 기존 파이프라인 유지, 다국어 질의는 **Agentic RAG**로 라우팅
- **Agentic RAG 구조** — 문서를 목차 기반 구조로 재구성하고, 목차와 설명 메타데이터를 LLM에 전달해 필요한 섹션을 스스로 판단·검색하도록 설계 — 언어에 무관하게 동작하는 검색 구조 확보
- **효과** — 다국어 답변 정확도 기존 수준 유지, 언어별 파이프라인 분기 없이 유지보수 단순화
- **트레이드오프** — 목차 판단 단계에서 토큰 사용량 증가, 한국어는 비용 효율을 위해 기존 파이프라인 병행 운영
