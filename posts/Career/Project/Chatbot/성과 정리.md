# 모델링

내가 말하고 싶은 부분은 문제에 맞는 적절한 파이프라인을 설계했다.

- 병원관련 질의 대답 상황
- 기본적인 RAG 파이프라인에서 제대로 검색과 답변이 되지 않는 문제
- 도메인 특화 파이프라인 개발
- 검색성능 70% 향상
- 40~50 -> 90~100 / 답변 정확도

**모델링 — 병원 도메인 특화 RAG 파이프라인 설계**

병원 도메인 질의응답 챗봇에서 기본 RAG 파이프라인의 검색 누락 문제를 도메인 특화 파이프라인 설계로 개선, **Hit Rate 70% 향상**

- 질의 앞단에 **분류 모델** 도입 — LLM 생성이 불필요한 질의(운영시간·주차 등 단순 정보, 상담원 연결, 예약·조회)를 고정답변으로 즉시 처리, 전체 질의의 약 50%가 분류 단계에서 처리되어 응답시간 **4~5초 → 1~2초**, 전체 평균 응답시간 약 **30% 단축**
- **Query Rewriting** — 대화 히스토리(Redis)를 반영해 질문을 독립적으로 재구성, 다턴 대화에서의 맥락 단절로 인한 검색 오류 해소
- **Query Decomposition + Keyword Extraction** — 복합 질의를 단위 질문으로 분해하고 병원 도메인 키워드를 추출, Semantic Search와 결합한 하이브리드 검색으로 검색 커버리지 향상
- LLM 생성 후 직접 검수한 평가셋(golden set) 기반으로 답변 정확도 **40~50점 → 90~100점** 향상
