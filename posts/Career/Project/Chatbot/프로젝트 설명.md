## 배경

병원 상담사들이 반복되는 고객 질의(운영시간, 가격, 예약 문의 등)에 시간을 소모하면서, 정작 필요한 상담 업무를 제대로 수행하지 못하는 상황이었습니다. FAQ 데이터셋을 기반으로 RAG(Retrieval-Augmented Generation) 파이프라인을 구축해 **자주 묻는 질문을 챗봇이 자동으로 처리**하도록 하고, 만족/불만족 피드백을 수집해 지속적으로 개선하는 방식으로 프로젝트를 운영했습니다.

이후 해외 채널로 서비스를 확장하면서, 한국어 특화 파이프라인이 다국어 환경에서 부딪히는 한계를 **Agentic RAG** 구조로 전환하여 해결했습니다.

## 문제 - 해결 - 성과

### 1. 반복되는 질의를 챗봇으로 대체해 기존 상담 40% 자동화

- **문제** : 상담사가 반복 질의에 시간을 뺏겨 본 업무에 집중 못함
- **해결** : FAQ 데이터셋 기반 RAG 파이프라인 구축 + 만족도 피드백 루프
- **성과** : 일평균 100~200건 반복 상담 자동화, 기존 상담 업무 **40% 자동화**

### 2. 기본 RAG의 검색 누락 문제를 도메인 특화 파이프라인으로 해결, Hit Rate 70% 향상

- **문제** : 맥락 단절, 복합 질의, 도메인 특화 단어(원장명 등)로 인한 검색 실패
- **해결** : Query Rewriting + Query Decomposition + Keyword Extraction + 임베딩 모델 평가/선정
- **성과** : **Hit Rate 70% 향상**, 검색 정확도 90% 이상
- 자세히 : [[Career/Project/Chatbot/RAG 파이프라인 설계|RAG 파이프라인 설계]]

### 3. LLM 생성이 불필요한 질의를 분류 모델로 고정답변 처리, 평균 응답시간 30% 개선

- **문제** : 답변 불가 질의에 LLM이 모호한 답변 생성 → 불만족 발생 / 단순 질의에도 LLM 호출로 응답 지연
- **해결** : RAG 파이프라인 앞단에 분류 모델 배치 → 고정답변 대상은 LLM 없이 즉시 응답
- **성과** : 응답시간 **4~5초 → 1~2초** (고정답변), 평균 **30% 단축**, 불만족도 **80% 이상 감소**
- 자세히 : [[Career/Project/Chatbot/질문 분류 및 응답시간 개선|질문 분류 및 응답시간 개선]]

### 4. 일평균 200건 대화기록 수동 검토를 LLM 기반 자동화로 전환, 검토 시간 83% 단축

- **문제** : Silent failure 감지를 위해 일평균 150~200건 대화기록을 수동 검토 (약 30분 소요)
- **해결** : RAGAS 기반 커스텀 평가지표 설계 + Airflow 자동화 + Slack 알림
- **성과** : 모니터링 시간 **30분 → 5분 (83% 단축)**, Silent failure 조기 감지 체계 구축
- 자세히 : [[Career/Project/Chatbot/모니터링 자동화|모니터링 자동화]]

### 5. 한국어 특화 파이프라인의 다국어 한계를 Agentic RAG로 해결, 유지보수 비용 대폭 감소

- **문제** : 한국어 특화 임베딩 / 키워드 추출 / 프롬프트가 다국어에 적용 불가 → 언어별 파이프라인 복제 시 유지보수 비용 폭증
- **해결** : Agentic RAG로 구조 전환 (LLM이 직접 검색 계획 수립 → Tool 호출 → 답변 생성)
- **성과** :
  - 언어별 파이프라인 N개 → **Agentic 단일 구조 1개**로 통합
  - 신규 언어 추가 공수 **약 80% 감소** (수주 → 수일, FAQ 문서만 추가)
  - 다국어 Hit Rate 한국어 대비 **90% 이상 수준 유지**
- 자세히 : [[Career/Project/Chatbot/글로벌 확장 - Agentic RAG|글로벌 확장 - Agentic RAG]]

## 기술 스택

**AI/ML**
- LLM : OpenAI GPT 시리즈 (질문 분류, 재정의, 쪼개기, 키워드 추출, 답변 생성, 평가)
- Embedding Model : 테스트셋 기반 비교 평가 후 선정
- Vector DB : Qdrant (시멘틱 서치 + 하이브리드 검색 + 멀티테넌시)

**Backend**
- FastAPI (RAG pipeline API)
- Redis (세션별 대화기록 관리)
- Docker, Amazon EC2

**Monitoring**
- Airflow (배치 평가 스케줄링)
- Google Spreadsheet (로그 수집)
- Slack (모니터링 알림)
- RAGAS 기반 커스텀 평가 지표
