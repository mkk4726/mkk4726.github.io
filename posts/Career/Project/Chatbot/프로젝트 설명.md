# 문제 - 해결 - 성과 - 도메인

> 고객상담에서 반복되는 질의를 챗봇으로 대체해 기존 상담 40% 자동화

>병원 도메인 질의응답 챗봇에서 기본 RAG 파이프라인의 검색 누락 문제를 도메인 특화 파이프라인 설계로 개선 해 Hit Rate 70% 향상

>병원 도메인 질의응답 챗봇에서 LLM 생성이 불필요한 질의를 분류 모델을 통한 고정답변 처리로 평균 응답시간 30% 개선

>일평균 200건의 대화기록을 담당자가 수동 검토하던 방식을 LLM 기반 자동화로 전환, 모니터링 담당자 검토 시간 20% 단축

>해외 채널로 확장시 한국어 특화 파이프라인의 다국어 성능 및 유지보수성의 한계로 다국어 질의에 대해 Agentic RAG로 검색 구조를 전환해 답변 품질 유지 및 유지보수비용 감소

>[!note]- 클로드가 다듬어준 버전
>1. 병원 도메인 반복 질의를 챗봇으로 대체해 **기존 상담 업무 40% 자동화**
>2. 기본 RAG 파이프라인의 검색 누락 문제를 도메인 특화 파이프라인(Query Rewriting · Query Decomposition · 하이브리드 검색) 설계로 개선, **Hit Rate 70% 향상 / 답변 정확도 40~50점 → 90~100점**  
>3. 질의 앞단에 분류 모델을 도입해 LLM 생성이 불필요한 질의(운영시간·예약·상담원 연결 등)를 고정답변으로 즉시 처리, **평균 응답시간 약 30% 단축**
>4. 일평균 200건의 대화기록 수동 검토를 LLM-as-judge 기반 자동 품질 평가(Context Sufficiency · Faithfulness)로 전환, **담당자 검토 시간 20% 단축**  
>5. 해외 채널 확장 시 한국어 특화 파이프라인의 다국어 성능 한계를 진단, 다국어 질의에 한해 목차 기반 Agentic RAG로 검색 구조를 전환해 **답변 품질 유지 및 유지보수 복잡도 감소**
