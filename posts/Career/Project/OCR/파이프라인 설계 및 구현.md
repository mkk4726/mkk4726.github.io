## 배경

프로젝트를 인수인계 받았을 때의 상황은 다음과 같았습니다.
- 전임자가 프로젝트를 진행하다가 중간에 떠난 상황
- 알 수 없는 에러로그가 쌓이고 있었고, OCR 모델의 정확도도 알 수 없음
- 어떤 문제를 해결하려 해도 **어디서 발생한 에러인지, 범위는 어디까지인지** 알 수 없는 구조

목표를 달성하기 위해서는 코드 구조를 개선하며 에러를 잡아나가야 했고, 이와 병행해 OCR 모델의 성능도 개선해야 했습니다.

## 1. 레거시 코드 분석 및 문제점 파악

코드 구조에는 심각한 문제가 있었습니다.
- 모든 기능이 하나의 `.py` 파일에 **절차지향적**으로 뒤섞여 있음
- 기본적인 디자인 패턴이나 역할별 모듈화 부재
- 에러 발생 시 원인 추적조차 쉽지 않았음
- 테스트 체계 전무 → 런타임 에러 사전 방지 장치 없음

이 상태에서는 어떤 개선도 리스크가 컸기 때문에, **코드 구조부터 바로잡는 것**을 최우선으로 두었습니다.

---

## 2. 객체지향적 코드 구조로 재설계

무자비하게 쌓인 코드들을 보면서, 머리로만 이해했던 1기능 1함수 원칙이나 디자인 패턴이 왜 필요한지를 마음으로 이해할 수 있었습니다. 여러 기능이 하나의 함수에 작성되어있고 타입힌트도 없는 코드를 이해하고 디버깅하는 과정은 굉장히 어려웠습니다.

하지만 이 과정에서 **이해하기 쉽고 디버깅이 쉬운 코드를 작성하기 위한 나름의 규칙**들을 세울 수 있었습니다.

1. 함수에는 꼭 하나의 기능만, 객체에는 하나의 역할만 부여하기
2. 객체 지향의 꽃은 추상화 — 입력과 출력만 신경쓸 수 있도록

이를 고려해 코드를 재작성했고 문제를 하나씩 고쳐나갈 수 있었습니다. 에러의 범위와 원인을 파악하기 쉬워졌고, 전체 문제를 작은 문제들로 쪼갤 수 있게 되었습니다.

## 3. 비동기 처리 구조 도입으로 속도 최적화

<figure>
<img src="./images/OCR_server_threadpool.png" alt="OCR Server 구조" width="80%" />
<figcaption>그림2. OCR Server 구조 (Client-Server 역할 재정의)</figcaption>
</figure>

처음 인수인계 받을 때 요구사항은 **동시에 여러 이미지를 처리하더라도 OCR 속도가 1초 미만**이어야 한다는 것이었습니다. Client 프로그램에서 Server 응답이 늦으면 이후 작업도 늦어지고, 이는 곧 고객 불편으로 이어지기 때문입니다.

접근법을 바꾼 출발점은 단순한 의문이었습니다.

> "Client에서 OCR 성공 여부를 굳이 확인할 필요가 있을까?"

Client 입장에서 필요한 것은 **이미지 송수신 성공 여부**뿐이었기에, 주고받을 성공여부의 의미를 재정의했습니다. Server 내부 역할을 분리해 **OCR 처리가 오래 걸리더라도 Client에는 영향을 미치지 않는 구조**로 재설계했습니다.

그 결과 평균 응답 속도를 **5초 → 1초**로 단축할 수 있었습니다.

## 4. 테스트 및 타입 검증 체계 구축

안정적인 파이프라인 운영을 위해 런타임 에러를 줄일 수 있는 방법을 고민했습니다.

선택한 방법은 **pytest 기반 unit test**와 **mypy를 통한 타입 검증**이었습니다.
- 작게 쪼개놓은 기능/역할들이 잘 수행되는지 unit test로 사전 확인
- 주고받는 값의 타입 불일치를 CI 과정에서 사전 검증
- 기능 추가/수정 시 발생할 수 있는 런타임 에러를 **사전에 방지**

이 과정을 통해 단순히 에러를 막는 것을 넘어서, 코드 품질과 서비스 안정성 자체를 보장하는 검증 체계를 갖출 수 있었습니다.

---

## 성과 및 배운 점

- **평균 응답 속도 5초 → 1초** (비동기 구조 적용)
- **에러 추적 가능한 OOP 구조**로 재설계 완료
- **CI 과정에 pytest + mypy 통합**으로 런타임 에러 사전 방지

객체지향적 코드 구조의 필요성과 그것이 가져다주는 유지보수성, 확장성, 안정성의 이점을 직접 체감했습니다. 특히 기능과 역할이 명확히 구분된 코드가 디버깅과 추가 개발의 복잡도를 크게 줄여주었고, 비동기 처리와 같은 최적화 포인트를 식별하는 데도 도움이 되었습니다.
