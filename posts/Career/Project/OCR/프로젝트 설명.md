## 배경

병원 검사결과는 검사장비마다 다른 포맷의 **이미지 파일**로 출력되어 곳곳에 흩어져 있었습니다. 환자 데이터를 기반으로 운영하는 서비스(시력 교정 수술 추천, 렌즈 사이즈 추천 등)의 관점에서 이는 큰 제약이었습니다.
- 담당자가 검사결과 이미지를 **수동으로 조회**해 확인 (건당 약 20초)
- 자동화된 데이터 수집 불가 → 서비스 운영 안정성 저하
- 검사결과를 한눈에 확인할 수 없는 구조

이 문제를 해결하기 위해 **검사장비에서 발생하는 이미지를 실시간으로 수신 → OCR로 텍스트 추출 → DB에 적재**하는 End-to-End 데이터 파이프라인을 설계하고 구현했습니다.

## 프로젝트 요약

- **일정** : 2025.03 ~ 2025.06
- **참여인력** : AI 1 (본인: AI 담당, 시스템 기획부터 구현까지 모든 과정)
- **서비스** : 병원에 이미지로 흩어져있던 검사결과를 OCR로 구조화해 한눈에 확인할 수 있도록 한 데이터 파이프라인
- **도메인** : 의료 데이터 자동화 / Computer Vision
- **규모** : 6가지 종류, 총 40~50대 검사장비 실시간 처리

## 문제 - 해결 - 성과

### [파이프라인]

> 병원 검사결과 이미지 수동 조회 프로세스를 OCR 파이프라인으로 자동화, 현장 담당자 인터뷰 및 프로세스 관찰 기반 측정 결과 데이터 확인 시간 95% 단축 (20초 → 1초/건)

- **문제** : 담당자가 수동으로 검사결과 이미지를 조회해 데이터를 확인 (건당 20초)
- **해결** : Client-Server 구조 재정의로 OCR 처리 지연이 사용자 경험에 영향을 주지 않는 비동기 파이프라인 구축
- **성과** : 데이터 확인 시간 95% 단축, 안정적인 데이터 수집 기반으로 서비스 운영 안정성 확보
- 자세히 : [[Career/Project/OCR/파이프라인 설계 및 구현|파이프라인 설계 및 구현]]

### [모델 정확도]

> 고정 레이아웃 특성을 활용해 Text Detection은 룰베이스, Text Recognition은 TrOCR로 설계하는 도메인 특화 파이프라인 구축 → 검증 세트 기준 Character-level Accuracy 100% 달성

- **문제** : 의료 서비스 특성상 99% 이상의 정확도가 요구되나, 초기에는 평가 데이터셋조차 없어 성능 검증 불가
- **해결** : 고정 레이아웃 특성을 활용해 Text Detection은 룰베이스로 단순화, Text Recognition은 TrOCR 오픈소스 모델 + 전처리/후처리 고도화
- **성과** : 검증 세트(200건) 기준 Character-level Accuracy 100% 달성
- 자세히 : [[Career/Project/OCR/OCR 정확도 개선|OCR 정확도 개선]]
