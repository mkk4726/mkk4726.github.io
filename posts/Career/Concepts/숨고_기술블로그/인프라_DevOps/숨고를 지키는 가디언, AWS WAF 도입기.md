# 숨고를 지키는 가디언, AWS WAF 도입기

## 원문

- 발행일: 2022.10.12
- 링크: https://soomgo.team/blog/posts/6673bb8e52107866fb86a7aa

## 한 줄 요약

숨고는 퍼블릭 ALB 앞단에 AWS WAF를 붙여 과도한 요청과 웹 공격을 차단하고, 로그 백업 자동화까지 포함한 보안 운영 체계를 구축했다.

## 왜 AWS WAF였나

- 숨고 인프라는 AWS 기반이며 EKS + ALB Ingress 구조를 사용 중이었다.
- AWS WAF는
  - 별도 설치 없이 바로 도입 가능
  - ALB와 조합 가능
  - 메트릭/로그 가시성이 좋음
  - 다른 AWS 보안 솔루션과 확장성이 높음
  이라는 장점이 있었다.

## 적용 방식

- 일반 요청은 WAF 규칙 검사를 통과해야 한다.
- 비정상 요청은 WAF에서 차단한다.
- WAF 메트릭과 로그를 기반으로 Slack 알람을 보낸다.
- 로그는 S3로 주기 백업해 비용과 분석 가능성을 함께 관리한다.

## 구현 포인트

- 우선 적용 대상을 Public ALB로 정했다.
- 스테이징에서 먼저 검증한 뒤 운영으로 올렸다.
- SQL Injection 탐지 같은 룰에는 custom response와 label match를 적극 활용했다.
- CloudWatch Logs 백업은 Docker + Kubernetes CronJob 방식으로 자동화했다.

## 효과

- 도입 전에는 과도한 크롤링이나 DDoS성 요청 때문에 `주 1회 수준`으로 가용성 저하가 있었다고 설명한다.
- 도입 후에는 Request Rate Limit 규칙이 과도한 요청 IP를 일시 차단해 가용성 확보에 도움을 줬다.
- 로그 백업 자동화만으로도 기존 대비 비용 약 `20% 이상` 절감했다.

## 인상적인 포인트

- 보안 도입기를 `룰 추가` 수준이 아니라 비용 산정, 스테이징 검증, 로그 백업까지 묶어서 다룬 점이 좋다.
- 숨고는 보안을 별도 팀의 영역이라기보다 DevOps 운영 체계 안에서 다루고 있다.
