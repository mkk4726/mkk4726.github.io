# Flutter가 React Native보다 낫습니다

## 원문

- 발행일: 2022.03.23
- 링크: https://soomgo.team/blog/posts/6673bb8f52107866fb86a7b0
- 비고: 원문 번역 글

## 한 줄 요약

이 글의 실질적 메시지는 `기술 비교보다 채용, 코드 공유, 조직 맥락이 더 중요하다`는 것이다.

## 핵심 주장

### 정말 중요한 기준

- 개발자 채용 풀
- 웹/백엔드/모바일 간 코드 공유
- 팀 간 지식 공유
- 기존 조직의 기술 스택과 연결성

이 관점에서는 React Native 쪽 손을 들어준다.

### Flutter가 강한 영역

- 개발자 경험(DX)
- Hot Reload와 CLI
- 업그레이드 경험
- 기본 성능
- 일관된 UI 렌더링
- 내장 네비게이션 등 즉시 사용성

### React Native가 강한 영역

- 채용 풀
- JavaScript/TypeScript 생태계
- 웹과의 코드/지식 공유
- 동적 업데이트(CodePush)
- 플랫폼 고유 경험에 더 자연스럽게 맞추기

## 숨고 맥락에서 읽을 포인트

- 숨고는 실제로 React Native, CodePush, 모바일 배포 자동화 관련 글이 많다.
- 따라서 이 번역 글은 단순 소개글이 아니라, 당시 팀이 `왜 React Native를 실용적인 선택으로 보는지`를 드러내는 텍스트로 읽힌다.

## 인상적인 포인트

- 제목은 도발적이지만 결론은 매우 현실적이다.
- 프레임워크 선택을 성능 싸움으로 좁히지 않고 `조직 설계 문제`로 보는 관점이 좋다.
