# LLM을 활용한 이미지 검수 시스템 개발

## 원문

- 발행일: 2025.10.20
- 링크: https://soomgo.team/blog/posts/68f5f17e4bb3dd93ca1f4ea0

## 한 줄 요약

숨고는 수작업 한계에 걸린 이미지 검수를 OpenAI `o4-mini`와 Batch API 기반 배치 파이프라인으로 자동화해, 하루 수천 건 규모의 검수를 전수 처리할 수 있게 만들었다.

## 배경

- 숨고 프로필에서 이미지는 신뢰 형성에 큰 영향을 준다.
- 하루 `3,000~6,000건`의 이미지가 업로드된다.
- 부적절한 이미지 업로드가 반복됐지만 수작업 검수는 물리적으로 한계가 있었다.

## 선택한 접근

### 모델 선택

- 비용, 안정성, 성능을 기준으로 OpenAI `o4-mini`를 선택했다.
- Prompt Caching과 Batch API를 활용하면 비용을 `50% 이상` 줄일 수 있다고 판단했다.

### 프롬프트 전략

- 파인튜닝 대신 프롬프트 고도화를 선택했다.
- 샘플 데이터 기반 테스트에서 `85% 이상` 정확도를 확보했다.

### 처리 방식

- 실시간 검수가 아니라 `하루 단위 배치 검수`로 설계했다.
- 업로드 이미지를 모아서 Batch API로 전송하고, 결과는 최대 24시간 내 회수한다.

## 구현 포인트

### 전송 형식

- `img_url` 대신 `base64 tokenize` 방식을 채택했다.
- 용량은 약 33% 늘지만, URL 노출과 인프라 접근 리스크를 줄일 수 있다고 봤다.

### 파일 처리

- JSONL 파일을 디스크에 쓰지 않고 메모리 ByteStream에서 생성해 업로드했다.
- 보안과 I/O 안정성을 동시에 고려한 선택이다.

### 배치 완료 확인

- 완료 시점을 정확히 알 수 없어 `1시간마다` 스케줄러가 상태를 점검한다.
- 완료되면 결과를 반영하고, 미완료면 다시 대기한다.

### 의존성 관리

- OpenAI 공식 패키지나 LangChain 대신 일단 `requests` 기반 직접 호출을 선택했다.
- 실험 단계에서 의존성을 최소화하려는 판단이다.

## 결과

- 이전에는 일 `6,000건` 규모 검수를 사람이 처리해야 했다.
- 자동화 이후 모든 작업물을 전수 검사할 수 있는 구조를 만들었다.
- 운영 인력은 반복 검수 대신 더 임팩트 있는 사용자 경험 개선 업무에 집중할 수 있게 됐다.

## 인상적인 포인트

- 실시간이 필요 없는 문제를 굳이 실시간으로 풀지 않고, `배치에 맞는 비용 구조`를 택한 점이 좋다.
- AI를 도입했지만 프레임워크 의존보다는 `문제에 맞는 최소 구현`을 우선했다.
- LLM 프로젝트를 기능 시연이 아니라 운영 프로세스 자동화에 연결한 사례다.

## 앞으로 볼 포인트

- LangChain 도입 여부
- Agent / MCP 기반 고도화
- 더 높은 성능의 최신 모델 적용 가능성
