# 모바일 앱 매주 배포하기 자동화하기

## 원문

- 원제: 모바일 앱 매주 배포하기: 자동화하기
- 발행일: 2021.04.26
- 링크: https://soomgo.team/blog/posts/6673bb8d52107866fb86a79c

## 한 줄 요약

숨고는 주간 모바일 배포를 지속하기 위해, 버전 변경과 태깅, 테스트용 배포, 마켓 배포를 `스크립트 + CircleCI + Slack 승인` 흐름으로 자동화했다.

## 전제

- 개발자가 작성한 코드는 두 번의 QA를 거친다.
  - 스쿼드 내부 QA
  - 통합 QA
- 테스트용 배포는 MS App Center를 활용했다.

## 자동화한 이유

- iOS와 Android를 함께 배포해야 하므로 반복 작업이 많다.
- 개발자가 직접 하면 한 번 배포에 `30~40분` 정도가 든다고 본다.
- 잦은 배포를 유지하려면 사람 손을 최대한 줄여야 했다.

## 자동화 구성

### 버전 변경 스크립트

- `npm run version:bump`로 버전 변경
- package.json, Android, iOS 설정 파일을 일관되게 수정
- git tag 생성까지 처리

### CircleCI 워크플로우

- `pre-release`: 테스트용 빌드 후 App Center 배포
- `release`: 마켓용 빌드 후 마켓 배포
- 태그가 push되면 시작되고, release / pre-release 여부에 따라 분기한다

### Slack 승인

- pre-release에서는 내부 테스터용 네이티브 앱 배포 여부를 Slack에서 승인
- release에서는 어떤 플랫폼을 배포할지 Slack에서 선택

## 결과

- 개발자는 배포 자체보다 개발에 집중할 수 있게 됨
- 새로 합류한 사람도 같은 절차로 쉽게 배포 가능
- 반복 작업이 일관된 방식으로 처리됨

## 인상적인 포인트

- 이 글은 모바일 CI/CD를 `QA 절차와 연결된 운영 흐름`으로 본다.
- 나중의 CodePush, AppCenter 대체 글과 연결해서 보면 숨고 모바일 팀이 일찍부터 배포 자동화를 핵심 과제로 다뤘다는 걸 알 수 있다.
