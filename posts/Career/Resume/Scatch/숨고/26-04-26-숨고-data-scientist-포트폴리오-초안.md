# 숨고 Data Scientist 포트폴리오 초안

## 포트폴리오 한 줄 메시지

검색 품질 개선, LLM 기반 운영 자동화, 의사결정 보조 모델링을 통해 서비스 지표를 개선해온 데이터 사이언티스트.

## 추천 구성

- 프로젝트 1: 고객상담 챗봇 개발 및 글로벌 확장
- 프로젝트 2: 렌즈 사이징 보조 AI 서비스
- 프로젝트 3: OCR 기반 데이터 수집 자동화

숨고 JD 기준으로는 1번이 가장 직접적이고, 2번이 문제 정의와 실험적 사고를 보여주며, 3번이 운영 가능한 시스템을 만든 경험을 보강합니다.

## 1. 검색 품질 개선과 LLM 운영 자동화를 통해 상담 업무의 40%를 자동화

`키워드`: Retrieval, Ranking, LLM, Monitoring, Routing, Agentic RAG

### 면접관 스캔용 요약

- 문제: 반복 상담이 많아 상담사가 핵심 업무에 집중하기 어려웠음
- 한계: 기본 RAG 구조는 다턴 맥락, 복합 질의, 도메인 고유명사 처리에 약했음
- 해결 1: Query Rewriting, Decomposition, Keyword Extraction 기반 검색 파이프라인 설계
- 해결 2: 질의 분류 단계를 추가해 생성이 불필요한 질문을 앞단에서 처리
- 해결 3: RAGAS 기반 자동 평가와 Airflow 배치 모니터링으로 silent failure 감지 자동화
- 해결 4: 글로벌 확장 시 Agentic RAG 구조로 전환해 언어별 파이프라인 복제를 피함
- 성과: 상담 40% 자동화, 검색 Hit Rate 70% 향상, 평균 응답시간 30% 단축, 모니터링 시간 83% 단축

### 문제

고객상담에서 운영시간, 비용, 예약 문의 같은 반복 질의가 많았고, 사람 상담사가 반복 응답에 시간을 쓰고 있었습니다. 초기 RAG 구조는 질문 맥락 복원, 복합 질의 처리, 병원 특화 용어 검색에서 누락이 잦아 검색 품질이 충분하지 않았습니다.

### 해결

검색 단계 앞에서 질의를 구조화하는 방식으로 접근했습니다.

- Redis 기반 대화 이력을 활용해 Query Rewriting 수행
- 복합 질문은 Query Decomposition으로 분해 후 검색 결과를 통합
- 도메인 키워드 추출과 메타데이터 필터를 결합한 하이브리드 검색 적용
- 운영시간, 주차, 상담원 연결 같은 질문은 분류 단계에서 즉시 처리
- 사람이 일일이 보던 대화 로그는 LLM 평가 자동화로 전환
- 다국어 확장 시에는 언어별 파이프라인을 복제하지 않고 Agentic RAG 단일 구조로 전환

### 성과

- 기존 상담의 40% 자동화
- 검색 Hit Rate 70% 향상
- 평균 응답시간 30% 단축
- 사용자 불만족도 80% 이상 감소
- 대화 로그 검토 시간 83% 단축
- 신규 언어 추가 공수 약 80% 절감

### 숨고와 연결되는 이유

- 숨고 JD의 `검색 모델 개선`, `GenAI/LLM 활용`, `비정형 데이터 분석`, `운영 효율화`와 가장 직접적으로 맞닿아 있는 사례
- 리뷰, 채팅, 검색 로그를 활용해 서비스 지표를 개선하는 접근을 설명하기 좋음

## 2. 점 추정의 한계를 불확실성 정량화 문제로 재정의해 실제 의사결정 품질을 개선

`키워드`: Problem Definition, Quantile Regression, Conformal Prediction, Experiment Design, Decision Support

### 면접관 스캔용 요약

- 문제: 모델 정확도 개선이 정체됐고, 현장은 예측값 자체를 신뢰하지 않았음
- 핵심 진단: 일부 중요한 정보가 관측되지 않아 점 추정만으로는 한계가 있었음
- 해결: Quantile Regression과 CQR로 예측 구간과 위험도를 제공하는 방향으로 문제 재정의
- 해결: 사용자 인터뷰 기반으로 확률 표현, 위험도 등급, 신뢰도 지표를 함께 설계
- 성과: 수술 안정성 10%p 개선, 재위치 건수 50% 감소, 현장 신뢰도 2.8 -> 4.2

### 문제

렌즈 사이즈 선택은 수술 결과에 큰 영향을 주지만, 현장에서는 경험과 휴리스틱에 의존하고 있었습니다. 모델 성능을 높여도 의료진이 결과를 신뢰하지 않으면 실제 사용으로 이어지지 않는 문제가 있었습니다.

### 해결

문제를 "정확한 값을 맞추는 일"이 아니라 "의사결정에 필요한 불확실성을 함께 제공하는 일"로 바꿨습니다.

- Quantile Regression으로 조건부 분위수 추정
- Calibration Set 기반 CQR 적용으로 신뢰수준을 보장하는 예측 구간 생성
- MAE 대신 Coverage와 Interval Width 중심의 평가지표 재설계
- 확률 분포, 정상 범위 내 위치 확률, 위험도 시각화를 사용자 화면에 반영
- 의료진 인터뷰를 통해 신뢰도와 활용성을 함께 측정

### 성과

- Vault 안정 비율 73% -> 83%
- 월별 렌즈 재위치 건수 20건 -> 10건
- 서비스 신뢰도 2.8 -> 4.2
- 정보 이해 용이성 2.3 -> 4.5

### 숨고와 연결되는 이유

- 숨고의 매칭/추천/최적화 문제도 단순 오프라인 성능보다 실제 사용자 의사결정과 거래 성공률을 함께 봐야 함
- 실험 설계, 지표 재정의, 이해관계자 설득 역량을 보여주기에 좋음

## 3. 데이터 수집 자동화를 통해 후속 모델링과 운영 안정성의 기반을 만든 OCR 시스템

`키워드`: Data Pipeline, OCR, Serving, Reliability, Async, Triton

### 면접관 스캔용 요약

- 문제: 검사결과가 이미지로 흩어져 있어 수동 조회에 시간이 많이 들었음
- 해결: 이미지 수집, OCR, DB 적재까지 하나의 파이프라인으로 재설계
- 해결: 고정 레이아웃 특성을 활용해 Detection은 단순화하고 Recognition 정확도에 집중
- 해결: 비동기 구조와 Triton 서빙으로 운영 안정성 확보
- 성과: 데이터 확인 시간 95% 단축, 응답속도 5초 -> 1초, 검증셋 기준 Character-level Accuracy 100%

### 숨고와 연결되는 이유

- 숨고에서도 모델 성능 이전에 `운영 가능한 데이터/모델 파이프라인`을 만드는 역량이 중요함
- MLOps 전 과정 경험과 실용적 시스템 설계 감각을 보여주는 보조 사례로 적합

## 숨고 맞춤 메시지

### 왜 숨고에 맞는가

- 숨고는 `고객-고수 연결`, `검색 품질`, `리뷰/채팅 기반 신뢰 형성`, `운영 효율화`가 핵심인 플랫폼
- 본인 경험 중 챗봇 프로젝트는 검색/비정형 데이터/LLM/운영 자동화라는 점에서 높은 적합도를 가짐
- 렌즈 사이징 프로젝트는 복잡한 현실 문제를 지표 재정의와 사용자 검증으로 풀어낸 경험을 보여줌
- OCR 프로젝트는 모델링뿐 아니라 실제 운영 가능한 파이프라인을 구축한 경험을 보강함

### 포트폴리오에서 강조할 톤

- 의료 도메인 세부 지식보다 `서비스 지표`, `검색 품질`, `운영 효율`, `사용자 신뢰`라는 범용 언어를 더 앞세우기
- "모델을 만들었다"보다 "무엇을 바꾸고 어떤 지표를 개선했는가"를 반복적으로 강조하기
- 추천/매칭 직접 경험이 약한 대신, 검색과 의사결정 보조 문제를 구조적으로 풀어낸 사례로 설득하기
