# 숨고 Data Scientist 자기소개서 초안

## 1. 지원 동기

저는 데이터 사이언스를 단순히 모델을 만드는 일보다, 서비스의 의사결정 구조를 더 나은 방향으로 바꾸는 일로 이해하고 있습니다. 그래서 새로운 회사를 볼 때도 "어떤 모델을 쓰는가"보다 "데이터와 알고리즘이 실제 사용자 경험과 거래 구조를 어떻게 바꾸는가"를 먼저 봅니다.

숨고는 고객과 고수를 연결하는 양면 시장 플랫폼으로, 매칭 성공률, 검색 품질, 리뷰와 채팅 기반의 신뢰 형성, 운영 효율화가 사업의 핵심이라고 이해했습니다. 기술블로그를 보면서도 단순히 새로운 기술을 도입하는 조직이 아니라, 리뷰 경험을 다시 설계하거나, LLM을 실무 운영 자동화에 붙이거나, 구조를 단순화해 더 큰 운영 문제를 해결하는 팀이라는 인상을 받았습니다. 이런 방식은 제가 일해온 방식과도 가깝습니다.

저 역시 의료 도메인에서 검색 품질 개선, LLM 기반 운영 자동화, 불확실성 정량화 기반 의사결정 보조 모델을 직접 설계하고 운영해왔습니다. 기술을 과시하기보다 실제 운영 흐름과 사용자 행동을 바꾸는 방식으로 문제를 풀어왔고, 이런 경험이 숨고 Data Scientist 역할과 가장 잘 맞는다고 판단해 지원했습니다.

## 2. 가장 관련성이 높은 경험

숨고 포지션과 가장 직접적으로 연결되는 경험은 고객상담 챗봇 프로젝트입니다. 당시 핵심 문제는 반복 상담이 많아 사람이 중요한 상담 업무에 집중하지 못한다는 점이었습니다. 처음에는 일반적인 RAG 구조로 접근했지만, 실제 운영 데이터를 분석해보니 문제는 단순히 "답변 품질이 낮다"는 수준이 아니었습니다. 다턴 대화에서 맥락이 끊기고, 복합 질의가 검색되지 않으며, 생성이 필요하지 않은 질문까지 모두 동일한 흐름으로 처리되면서 속도와 만족도가 함께 떨어지고 있었습니다.

저는 이 문제를 검색 이전 단계에서부터 다시 설계했습니다. Query Rewriting, Query Decomposition, Keyword Extraction을 도입해 검색 품질을 끌어올렸고, 생성이 필요 없는 질문은 질의 분류 단계에서 바로 처리하도록 구조를 바꿨습니다. 또 챗봇 특유의 silent failure를 줄이기 위해 RAGAS 기반 자동 평가와 Airflow 배치 모니터링을 구축했습니다. 그 결과 상담의 40%를 자동화했고, 검색 Hit Rate를 70% 높였으며, 평균 응답시간을 30% 줄이고 대화 검토 시간을 83% 단축할 수 있었습니다.

이 경험에서 제가 배운 것은, 서비스 문제는 모델 하나로 풀리지 않는다는 점입니다. 좋은 데이터 사이언티스트는 모델을 선택하는 사람에 그치지 않고, 어떤 문제를 어떤 구조로 풀어야 하는지 정의하고, 운영 지표가 실제로 개선되는지 끝까지 확인해야 한다고 생각합니다. 숨고에서도 매칭, 검색, 채팅, 리뷰처럼 여러 접점의 문제를 같은 관점으로 풀 수 있다고 생각합니다.

## 3. 제가 일하는 방식

제가 반복적으로 보여준 강점은 세 가지입니다.

첫째, 문제를 다시 정의합니다. 렌즈 사이징 프로젝트에서는 점 추정 성능 개선이 정체됐을 때, 더 좋은 예측값을 만드는 대신 불확실성을 함께 제공하는 방식으로 문제를 바꿨습니다. 그 결과 의료진이 실제로 활용할 수 있는 예측 구간과 위험도 지표를 만들 수 있었고, 수술 안정성과 사용자 신뢰도를 함께 개선했습니다.

둘째, 서비스 지표와 모델 지표를 분리해서 봅니다. 오프라인 성능이 좋아도 사용자가 신뢰하지 않거나 운영 흐름에 맞지 않으면 실제 임팩트로 이어지지 않는다는 점을 여러 프로젝트에서 확인했습니다. 그래서 저는 모델 성능뿐 아니라 응답시간, 사용성, 신뢰도, 반복 업무 절감 같은 지표를 함께 설계하고 봅니다.

셋째, 운영 가능한 구조를 만듭니다. OCR 프로젝트에서는 이미지 수집, OCR, DB 적재까지 하나의 파이프라인으로 연결했고, 비동기 구조와 Triton 서빙을 적용해 실제 운영 환경에서 안정적으로 동작하도록 만들었습니다. 모델링과 운영을 분리해서 보지 않고, 끝까지 책임지는 방식으로 일해왔습니다.

## 4. 숨고에서 기여하고 싶은 부분

입사하게 된다면 가장 먼저 기여하고 싶은 부분은 검색과 비정형 데이터 활용 영역입니다. 숨고는 고객의 의도를 정확히 이해하고 적절한 고수, 서비스, 콘텐츠를 연결하는 것이 핵심이고, 그 과정에서 검색 로그, 채팅, 리뷰 데이터는 매우 중요한 자산이라고 생각합니다. 저는 이 데이터를 활용해 검색 품질을 개선하고, LLM을 운영 효율화와 의사결정 지원에 연결하는 일에 빠르게 기여할 수 있습니다.

중장기적으로는 추천과 매칭 문제에도 기여하고 싶습니다. 직접적인 O2O 추천 경험은 아직 많지 않지만, 사용자 의사결정 문제를 정량화하고 지표를 다시 설계해 실제 행동 변화를 만든 경험은 분명한 강점이라고 생각합니다. 숨고처럼 양면 시장 구조가 있는 서비스에서는 모델 성능뿐 아니라 거래 성공률, 신뢰 형성, 운영 효율까지 함께 보는 시각이 중요하다고 생각하며, 그 관점에서 기여하겠습니다.

## 마무리 문장

저는 짧은 기간 동안에도 문제 정의부터 모델 운영까지 넓은 범위를 직접 맡아왔고, 기술을 실제 서비스 임팩트로 연결하는 일을 가장 중요하게 생각해왔습니다. 숨고에서도 검색, 매칭, LLM 활용 영역에서 데이터 기반으로 더 나은 연결 경험을 만드는 데 기여하고 싶습니다.
