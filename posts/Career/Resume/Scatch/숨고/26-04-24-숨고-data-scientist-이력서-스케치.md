# 숨고 Data Scientist 이력서 초안

## 상단 요약

서비스 지표 개선을 목표로 문제 정의, 실험 설계, 모델링, 배포 및 모니터링까지 연결하는 데이터 사이언티스트입니다. 의료 도메인에서 검색 품질 개선, LLM 기반 운영 자동화, 불확실성 정량화 기반 의사결정 보조 모델을 직접 설계하고 운영해왔습니다. 기술 자체보다 실제 사용자 경험과 운영 지표를 어떻게 개선할지에 집중합니다.

## 핵심 역량

- 문제 정의부터 모델 운영까지 End-to-End로 수행한 경험
- 검색 품질 개선, LLM 기반 비정형 데이터 활용, 운영 자동화 경험
- 서비스 지표와 모델 지표를 분리해 설계하고 검증하는 업무 방식
- 도메인 전문가, 운영 조직과 협업하며 요구사항을 구조화하는 커뮤니케이션 역량
- 통계적 근거를 바탕으로 방법론을 선택하고, 과도한 복잡성을 피하는 실용적 설계 감각

## 경력

### 비쥬웍스 | Data Scientist
- 기간: 2024.04 ~ 현재
- 산업: 의료 AI / 헬스케어
- 역할: 데이터 파이프라인 구축, 모델링, 서빙, 운영 모니터링, 서비스 개선

#### 1. 고객상담 챗봇 개발 및 글로벌 확장
- 기간: 2024.11 ~ 2025.02, 2025.08 ~ 2025.11
- 역할: AI 담당
- 반복 질의를 자동화하는 RAG 기반 상담 챗봇을 설계해 기존 상담의 40%를 자동화
- Query Rewriting, Query Decomposition, Keyword Extraction 기반 검색 파이프라인을 설계해 RAG 검색 Hit Rate를 70% 향상
- 질의 분류 단계를 추가해 전체 질의의 약 50%를 선처리하고 평균 응답시간을 30% 단축, 불만족도를 80% 이상 감소
- RAGAS 기반 자동 평가와 Airflow 배치 모니터링을 구축해 일평균 150~200건 대화 검토 시간을 83% 단축
- 다국어 확장 시 언어별 파이프라인 복제 대신 Agentic RAG 구조로 전환해 신규 언어 추가 공수를 약 80% 절감

#### 2. 렌즈 사이징 보조 AI 서비스 개발
- 기간: 2025.11 ~ 현재
- 역할: AI 담당
- 수술 전 환자 정보로 수술 후 핵심 지표를 예측하는 의사결정 보조 서비스를 개발해 Vault 안정 비율을 73%에서 83%로 개선
- 점 추정의 한계를 Quantile Regression + Conformal Prediction 기반 예측 구간 문제로 재정의해 3개 임상 지표의 위험도를 정량화하는 파이프라인 구축
- 사용자 인터뷰와 의료진 피드백을 반영해 확률 표현, 위험도 시각화, 신뢰도 지표를 설계하고 현장 사용자 신뢰도를 2.8점에서 4.2점으로 개선
- 학습-추론 일관성을 위한 데이터 파이프라인, 피처 관리, 모니터링 체계를 구축해 베이스라인 대비 예측 성능 30% 향상

#### 3. OCR 기반 검사결과 자동화 시스템 구축
- 기간: 2025.03 ~ 2025.06
- 역할: AI 담당
- 병원 검사결과 이미지 수집부터 OCR 추론, DB 적재까지 End-to-End 파이프라인을 구축해 데이터 확인 시간을 95% 단축
- 고정 레이아웃 특성을 활용해 Detection은 룰 기반으로 단순화하고 Recognition은 TrOCR 중심으로 고도화해 검증셋 200건 기준 Character-level Accuracy 100% 달성
- 비동기 Client-Server 구조와 Triton 기반 서빙을 구성해 평균 응답속도를 5초에서 1초로 개선

## 숨고 JD와 연결되는 포인트

- 검색 모델 개선: 챗봇 RAG 검색 품질 개선 경험이 가장 직접적으로 연결됨
- GenAI / LLM 활용: 비정형 질의 처리, 로그 평가, 운영 자동화 경험 보유
- 추천 / 매칭에 필요한 사고방식: 단순 성능 개선보다 사용자 의사결정과 서비스 지표를 함께 보는 방식에 강점
- 실험 설계 및 설득: 인터뷰, 전후 비교, 사용자 반응 지표 설계 경험 보유
- MLOps 전 과정 경험: 데이터 수집, 학습, 배포, 모니터링까지 반복적으로 수행

## 작성 메모

- 숨고는 `경력 3년+` 조건이 있어 연차는 정직하게 두되, 짧은 기간 동안 맡은 책임 범위를 전면에 두는 것이 좋음
- 이력서 본문에서는 의료 도메인 자체보다 `검색 품질`, `운영 자동화`, `서비스 지표 개선`, `실험적 사고`를 더 강하게 표현하는 편이 유리함
- 추천, Fraud, Bidding 직접 경험은 과장하지 말고 검색/LLM/실험 설계 중심으로 정렬하는 것이 안전함
