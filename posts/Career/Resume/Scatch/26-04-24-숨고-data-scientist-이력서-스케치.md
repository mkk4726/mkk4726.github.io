# 이력서 초안 (숨고 - Data Scientist)

## 포지션 핏 한 줄
- 문제 정의부터 실험 설계(A/B), 모델 배포·모니터링까지 연결하는 제품형 데이터 사이언티스트 포지셔닝.

## 상단 소개 문구 초안
서비스 지표 개선을 목표로 문제 정의, 실험 설계, 모델링, 배포 및 모니터링까지 주도하는 데이터 사이언티스트입니다. 통계적 근거를 기반으로 방법론을 선택하고, ML/LLM을 실제 운영 흐름에 맞게 적용해 사용자 경험과 비즈니스 지표를 함께 개선해왔습니다.

## 핵심 경력 포인트 (JD 매핑)
- 매칭/검색/추천에 바로 연결 가능한 분석 역량
  - 사용자 피드백과 로그를 정량 분석해 병목 원인을 규명하고 개선 우선순위를 구조화.
  - 모델 품질 지표뿐 아니라 사용자 반응 지표까지 포함해 실험 결과를 해석.
- 실험 설계 및 검증 중심 업무 방식
  - 가설 설정 -> 지표 정의 -> 실험 설계 -> 결과 검증의 사이클을 반복.
  - A/B 테스트와 유사한 방식으로 변경 전후 성과를 비교해 의사결정 지원.
- MLOps 전 과정 경험
  - 데이터 파이프라인 구축, 학습-추론 일관성 관리, 운영 모니터링 자동화 경험.
  - Airflow 기반 주기 실행 및 품질 점검 체계 운영.
- LLM 기반 비정형 데이터 활용
  - 비정형 텍스트(질문/피드백)에서 패턴을 추출하고 분류 체계를 도입해 서비스 개선에 활용.
  - LLM을 생성 도구를 넘어 평가/분석 보조 연산기로 활용.
- 통계/ML 이론 기반 방법 선택
  - 단순 성능 개선이 아닌 불확실성 정량화, 지표 재설계 등 문제 특성에 맞는 접근 적용.

## 기술 스택 강조
- Python, SQL, Airflow, 실험 설계/지표 분석
- ML 모델링, LLM 활용 분석, 운영 모니터링 자동화

## 자기소개서/면접용 포인트
- "어떤 모델을 썼는가"보다 "어떤 문제를 어떻게 정의하고 검증했는가"를 앞세우기
- 서비스 지표와 모델 지표를 분리해 관리한 경험 강조
- 장기적으로 Fraud/Optimal Bidding 등 확장 과제에도 바로 기여 가능한 분석 프레임 제시
