# 이력서 초안 (북아이피스/쏠북 - ML Engineer)

## 포지션 핏 한 줄
- OCR + LLM Agent + 실험 중심 개선 사이클을 직접 돌려본 경험을 전면에 배치.

## 상단 소개 문구 초안
OCR, 전통 ML/DL, LLM을 결합한 하이브리드 파이프라인을 설계하고 운영해 온 ML 엔지니어입니다. 문제 정의부터 프로토타입, 운영 개선까지 빠른 실험-검증 사이클을 주도하며, 복잡한 비정형 데이터를 서비스 가능한 구조로 전환하는 데 강점이 있습니다.

## 핵심 경력 포인트 (JD 매핑)
- Hybrid 생성/추론 파이프라인 설계
  - 전통적 ML 접근과 LLM 기반 접근을 문제 특성에 맞게 결합하는 설계 경험.
  - 모델 정확도뿐 아니라 비용, 지연시간, 유지보수성을 함께 고려해 아키텍처 선택.
- OCR/비정형 문서 구조화 경험
  - OCR 파이프라인 구축 및 고도화 경험을 바탕으로 비정형 문서를 구조화 데이터로 변환.
  - 데이터셋 구축-평가-에러분석 루프를 통해 품질 개선 체계화.
- Agentic LLM 활용 역량
  - Agentic RAG 구조 설계 경험과 함께, LLM 호출 전략을 조정해 품질/비용 균형 확보.
  - 프롬프트 기반 분류/평가 자동화로 운영 효율 개선.
- 주도적 문제 해결 방식
  - 문제 정의, 우선순위 설정, 실험 설계, 결과 반영까지 독립적으로 수행한 프로젝트 경험.
  - 최신 방법론을 탐색하되 과도한 복잡성을 피하고 실서비스 적용 가능성 중심으로 판단.
- 확장형 운영 기반
  - API 기반 서빙 구조 및 자동화 워크플로우 운영 경험.
  - 모니터링 지표를 직접 설계해 성능 저하를 조기 감지.

## 기술 스택 강조
- Python, PyTorch, OpenCV, OCR 파이프라인
- LLM/Agentic RAG, 프롬프트 기반 분류/평가
- FastAPI, Docker, Airflow, AWS 기반 운영 경험

## 자기소개서/면접용 포인트
- "콘텐츠 생성" 과제를 "품질-속도-운영성" 3축으로 풀어낸 경험 강조
- 교육 도메인 미경험은 문서 구조화/OCR/LLM 적용 역량으로 빠르게 전이 가능함을 설명
- 시스템 디자인 인터뷰 대비: 파이프라인 단계별 병목과 개선 우선순위를 사례 중심으로 정리
